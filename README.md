API Log Monitoring Mini Project

Overview

This project demonstrates a simplified API monitoring system built using Python. It simulates how organizations monitor API usage, availability, and performance by analyzing log files generated from API calls.

In real production environments, monitoring platforms such as Splunk, ELK Stack, Datadog, Prometheus, and Grafana analyze API logs and metrics to detect anomalies like:
	•	High API error rates
	•	Traffic spikes
	•	Increased response latency
	•	Service outages

This project replicates the core monitoring logic used in those systems using lightweight Python scripts suitable for learning and portfolio demonstration.

⸻

Problem This Project Solves

Modern systems rely heavily on APIs. Monitoring API health is critical to ensure services remain available and performant.

Organizations typically monitor:
	•	Usage Metrics – number of requests per endpoint
	•	Availability Metrics – HTTP error rates
	•	Performance Metrics – response latency

This project demonstrates how such metrics can be derived from API access logs.

Components

1. API Collector Script

File: api_collector.py

This script simulates API traffic by calling public APIs and logging the results.

Data Collected

Each API request records:
	•	Timestamp
	•	HTTP method
	•	Endpoint
	•	Status code
	•	Response latency (milliseconds)

Example Log Entry

2026-03-04T20:10:01+00:00 GET https://api.github.com 200 120

Log Format

timestamp method endpoint status_code latency_ms

These logs simulate API access logs generated by production services.

⸻

2. Monitoring Script

File: monitor.py

This script performs batch log analysis and calculates monitoring metrics.

Usage Monitoring
	•	Total API requests
	•	Requests per endpoint

Availability Monitoring
	•	HTTP error rate
	•	Detection of 4xx and 5xx responses

Performance Monitoring
	•	Average response latency
	•	Maximum response latency

How to Run the Project

1. Clone the Repository

git clone <repo-url>
cd api-log-monitor-project

2. Create Virtual Environment

python3 -m venv venv
source venv/bin/activate

3. Install Dependencies

pip install -r requirements.txt

4. Generate API Logs

Run the collector multiple times to generate sample traffic.

python api_collector.py

5. Analyze Logs

python monitor.py

The script will generate a monitoring report based on collected logs.
