API Log Monitoring Mini Project

Overview

This project demonstrates a simplified API monitoring system built using Python. It simulates how organizations monitor API usage, availability, and performance by analyzing log files generated from API calls.

In production environments, monitoring platforms such as:
	‚ÄĘ	Splunk
	‚ÄĘ	ELK Stack
	‚ÄĘ	Datadog
	‚ÄĘ	Prometheus
	‚ÄĘ	Grafana

	‚ÄĘ	High API error rates
	‚ÄĘ	Traffic spikes
	‚ÄĘ	Increased response latency
	‚ÄĘ	Service outages

This project reproduces the core logic of those monitoring systems using lightweight Python scripts suitable for learning, demonstrations, and DevOps portfolios.

Problem This Project Solves

Modern systems rely heavily on APIs. Monitoring API health is critical to ensure services remain available, reliable, and performant.

Organizations typically monitor three major metric categories:

Usage Metrics
	‚ÄĘ	Number of requests per endpoint
	‚ÄĘ	Traffic patterns

Availability Metrics
	‚ÄĘ	HTTP error rates
	‚ÄĘ	Detection of 4xx and 5xx responses

Performance Metrics
	‚ÄĘ	Average response latency
	‚ÄĘ	Maximum response latency

This project demonstrates how these metrics can be derived from API access logs.

Core Components

1. API Collector Script

File: api_collector.py

This script simulates API traffic by calling public APIs and logging responses.

Data Collected

Each request records:
	‚ÄĘ	Timestamp
	‚ÄĘ	HTTP method
	‚ÄĘ	Endpoint
	‚ÄĘ	Status code
	‚ÄĘ	Response latency (ms)

Example Log Entry

2026-03-04T20:10:01+00:00 GET https://api.github.com 200 120

Log Format

timestamp method endpoint status_code latency_ms

These logs simulate API access logs generated by production services.

Running the Project

1. Clone Repository

git clone <repo-url>
cd api-log-monitor-project

2. Create Virtual Environment

python3 -m venv venv
source venv/bin/activate

3. Install Dependencies

pip install -r requirements.txt

4. Generate API Logs

python api_collector.py

Run multiple times to simulate traffic.

5. Analyze Logs

python monitor.py

The script generates a monitoring report.
